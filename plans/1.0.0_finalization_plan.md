# 1.0.0 finalization plan

This document references the sub-plans for the tm1_git_py project in the recommended execution order. Each sub-plan lives in the `plans/` folder.

---

## 1. Separate dev/test dependencies

**Sub-plan:** [refactor_separate_dev_test_dependencies.md](refactor_separate_dev_test_dependencies.md)

Split runtime and test/dev dependencies so that installing the package as a dependency does not pull in pytest, testcontainers, etc. Runtime-only in `requirements.txt`; test/dev in `requirements-dev.txt` and the `[dev]` extra. CI and README updated accordingly.

*Status: Implemented (summary document).*

---

## 2. Eliminate tm1_bedrock_py (scope limit)

**Sub-plan:** [refactor_eliminate_tm1_bedrock_py.md](refactor_eliminate_tm1_bedrock_py.md)

Remove the `tm1_bedrock_py` dependency from requirements, pyproject.toml, setup.py, and README to limit initial scope. It will be re-added in a later release when the cube-recreate/data-copy feature is enabled.

*Status: Implemented (summary document).*

---

## 3. Changeset apply and status tracking

**Sub-plan:** [rework_changeset_apply_and_status_tracking.md](rework_changeset_apply_and_status_tracking.md)

Rework apply and status so that: `changeset_name` stays on the changeset (current init kept, non-optional `str`); `execution_id` is generated during apply as `changeset_name + timestamp` (yyyyMMddHHmmss); `status_dir` is removed and apply returns status in the response (caller serializes if needed); `last_execution_id` is removed from Changeset.

*Status: Plan only.*

---

## 4. Compare and apply CLI

**Sub-plan:** [main_py_compare_and_apply_cli.md](main_py_compare_and_apply_cli.md)

Add `compare` and `apply` CLI commands to main.py using subparsers. Compare: two model folders → output changeset file (YAML/JSON). Apply: changeset file + server → apply to TM1; optional status-dir/execution-id in CLI for caller to persist status.

*Status: Plan only.*

---

## 5. Inconsistencies and refactors

**Sub-plan:** [codebase_inconsistencies_and_refactors.md](codebase_inconsistencies_and_refactors.md)

Address inconsistencies (TM1py import style, type hints, duplicate path parsing, dead code, typos, dual setup.py/pyproject.toml) and refactor opportunities (shared path normalization, exporter Element type, error handling in deserializer/exporter). Use as a backlog; implement in order of priority.

*Status: Plan/backlog only.*

---

## Execution order

| Order | Sub-plan | Purpose |
|-------|----------|--------|
| 1 | Separate dev/test dependencies | Clean dependency boundary before other work. |
| 2 | Eliminate tm1_bedrock_py | Reduce scope; re-add later. |
| 3 | Changeset apply and status | Simplify apply API and status handling before CLI. |
| 4 | Compare and apply CLI | Expose compare/apply via main.py. |
| 5 | Inconsistencies and refactors | Incremental cleanup and improvements. |

Plans 1 and 2 are already done (summaries). Plans 3, 4, and 5 are to be implemented in this order when ready.
